Project Name: Ascendency Core Concept: The world's first free "Looksmaxing" RPG. Vibe: "Solo Leveling" / Futuristic Anime (Dark mode, Neon Blue, System Interfaces).

1. üì± User Flow & Onboarding Logic
Account Creation: Immediate requirement. Email/Password or Social Auth (Supabase Auth).

The "Hunter" Survey:

Inputs: Height, Age, Weight, Ethnicity, Lifestyle Habits (Smoking, Sleep, Water intake).

Data use: Used to calibrate the recommendation engine (e.g., specific advice for specific ethnicities).

The Face Scan (Biometric Ingestion):

Input: Camera capture (Selfie).

Security: Display BIPA/Privacy Consent Popup before camera opens.

Processing: Image sent to backend (AI Vision model) to analyze features.

2. üß† The "System" Logic (Backend Analysis)
The Core Algorithm: The backend must process the Survey + Scan to generate a "Face Potential Score" (0-100) and a "Looksmaxing Plan."

Target Features to Detect (The "Flaws"): The AI must explicitly look for and tag:

Face Fat / Face Bloat

Eyebrow Shape (Weak/Bad)

Acne / Skin Texture

Hair Style Suitability

Skin Tone (Pale/Dullness)

Teeth (Yellow/Crooked)

Eye Bags / Dark Circles

Sclera Color (Yellowish/Red)

The "ROI" Sorting Engine: Recommendations must be sorted by Return on Investment.

Logic: Priority Score = (Aesthetic Impact) / (Time & Effort Cost)

Example: Whitening teeth strips (High Impact / Low Effort) = Rank 1.

Example: Jaw surgery (High Impact / Extreme Effort) = Rank 10.

Tab 1: The Report (Home)

Header: User's "Face Potential Score" and Current Level.

Body: Scrollable list of "Quests" (Recommendations).

Organization: Sorted by Priority/ROI (Immediate fixes first).

Detail View: Clicking a specific issue (e.g., "Face Bloat") opens a drawer with:

Tab 2: Progress (The Gallery)

Layout: "Before & After" slider comparison.

Constraint: User must take a monthly selfie.

UI Helper: When taking a new selfie, overlay the original selfie as a semi-transparent "ghost" guide to ensure same angle/lighting.

Tab 3: Scan & Calibration

Function: Allow user to retake Survey or Face Scan.

Logic Gate: "One Free Redo." If the user messes up the first scan, they get 1 free retry. Subsequent retries are locked (Premium feature).

Tab 4: Tasks / Calendar (The Quest Log)

View A (Daily): Checklist of daily habits generated from the Report (e.g., "Mewing 30 mins", "Skincare Routine").

View B (Calendar): Monthly view of adherence.

Gamification: Completing tasks grants XP. XP levels up the user's Avatar.

Tab 5: Profile (Hunter Info)

Display: User Avatar, Level, Settings.

Consultations: "Meeting Scheduler" button integration (WebView or API link to Calendly) for 1-on-1 coaching.
